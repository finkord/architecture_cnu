Payment Service

Functional Requirements:
FR-5.1	As a user, I want to pay for a course online, so that I can access premium content.
FR-5.2	As a student, I want to receive immediate access to a course after successful payment, so that I can start learning without delay.
FR-5.3	As a user, I want to view my payment history, so that I can track transactions.

Non-functional Requirements:
NFR-5.1	The payment service should process payments within 5 seconds, so that users do not experience delays.
NFR-5.2	The payment service should be available 99.9% of the time, so that users can make payments whenever they need to.
NFR-5.3	The payment service should be secure, so that users' payment information is protected.
NFR-R1	Enrollment and payment operations must be atomic.
NFR-S4	Sensitive data (payments) hidden in logs and responses.
